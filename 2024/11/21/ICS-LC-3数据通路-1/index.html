LC-3 数据通路

如图，展示的是 LC-3 数据通路，即 LC-3 的硬件设计架构。通过这个图可以理解指令如何被取出、解码和执行。

1. 总线（Bus）

  • 图片顶部横向的黑线是 16 位宽的共享总线，数据通过这个总线在不同组件之间传输。
  • 任何一个组件要向总线发送数据，必须启用对应的 Gate 信号（例如 GateMDR，GatePC 等）。

2. 寄存器组（Reg File）

  • 寄存器组（右上角的 REG FILE）是 LC-3 中的通用寄存器（R0–R7），每个寄存器 16 位，用于存储操作数或结果。
  • SR1 和 SR2 是两个源寄存器，SR2MUX 决定 SR2 的来源：
    • 可以直接使用寄存器的值。
    • 也可以从指令的立即数（通过 SEXT）中获取数据。
  • DR（Destination Register）是目标寄存器，存放结果。

3. 程序计数器（PC）

  • PC（Program Counter） 位于右上角中间，用于存储下一条指令的内存地址。
  • PC 的更新路径：
    • PCMUX 选择 PC 的更新来源：
      1. PC + 1：正常取下一条指令。
      2. 新的跳转地址（如分支指令）。
      3. 异常情况的处理地址。
    • GatePC 将 PC 的值送到总线，用于访问内存。

4. 指令寄存器（IR）

  • IR（Instruction Register） 位于中间偏左，保存当前从内存中取出的指令。
  • LD.IR 信号控制 IR 是否加载新的指令。

5. 存储器单元（Memory）

  • LC-3 的存储器在图的底部：
    • MAR（Memory Address Register）: 存储需要访问的内存地址。
    • MDR（Memory Data Register）: 存储从内存读取或即将写入内存的数据。
    • LD.MAR 和 LD.MDR 控制是否加载新数据。
    • MEM.EN、R.W 信号控制内存的操作：启用访问、读/写切换。
  • 内存与总线通过 GateMDR 和 GateMARMUX 进行交互。

6. 算术逻辑单元（ALU）

  • ALU 位于右下角，用于执行算术或逻辑操作。
  • 输入：
    • A 和 B：从寄存器或指令中获得操作数。
    • SR2MUX 决定第二个操作数是寄存器值还是指令中的立即数。
  • 输出：
    • 运算结果可以写回寄存器，或者更新条件码寄存器（CC）。
  • GateALU 控制 ALU 的输出是否送到总线。

7. 条件码寄存器（CC）

  • 位于 ALU 输出旁边，用于存储上次运算结果的状态：
    • N（负）、Z（零）、P（正）。
  • LD.CC 控制条件码是否更新。

8. 多路选择器（MUX）

LC-3 中使用了大量的多路选择器（MUX）来选择数据来源，以下是几个关键的 MUX：

  1. MARMUX（Memory Address MUX）：
    • 决定存储器地址来源是：
      • 来自 ZEXT（零扩展的立即数）或
      • ADDR1MUX 和 ADDR2MUX 计算出的地址。
  2. ADDR1MUX：
    • 决定计算地址时，基地址是：
      • PC 或
      • 寄存器值。
  3. ADDR2MUX：
    • 决定偏移量来源：
      • 指令中的 5 位、6 位或 9 位立即数（符号扩展 SEXT）。
  4. PCMUX：
    • 决定下一条 PC 的来源（见 PC 部分）。
  5. SR2MUX：
    • 决定 ALU 第二个操作数的来源（寄存器或立即数）。

9. 输入输出

  • LC-3 的输入和输出接口位于底部右侧：
    • INPUT 和 OUTPUT 用于与外部设备交互。
    • 数据通过总线传递到这些接口。

10. 控制单元（有限状态机Finite State Machine）

  • 控制单元位于图的中央，核心是一个有限状态机（FSM），通过指令解码生成控制信号。
  • 控制信号包括：
    • LD 信号：控制寄存器是否加载新数据（如 LD.PC，LD.MDR）。
    • Gate 信号：控制哪些模块将数据送到总线（如 GatePC，GateMDR）。
    • ALUK 信号：控制 ALU 执行何种操作（如加法、与等）。

指令执行的流程

以下是指令执行的大致过程：

  1. 取指（Fetch）：
    • PC 的值通过总线送到存储器（MAR 和 MDR），读取指令存入 IR。
    • PC 加 1。
  2. 解码（Decode）：
    • 控制单元解析 IR 的内容，生成控制信号。
  3. 执行（Execute）：
    • 如果是算术指令，寄存器的值送入 ALU，计算结果写回寄存器。
    • 如果是内存指令，地址通过 MARMUX 计算，访问存储器。
  4. 写回（Write Back）：
    • 将执行结果存入目标寄存器（DR）。

含义：将寄存器 R1 和 R3 的值相加，并将结果存入 R2 中。

指令格式

ADD 指令的二进制格式如下：

  • **0001**：操作码（Opcode），表示 ADD 指令。
  • **DR**（Destination Register）：目标寄存器（这里是 R2）。
  • **SR1**（Source Register 1）：第一个操作数寄存器（这里是 R1）。
  • **SR2**（Source Register 2）：第二个操作数寄存器（这里是 R3）。

执行过程

LC-3 中的指令执行分为 6 个阶段：取指（Fetch）、解码（Decode）、取操作数（Fetch Operands）、执行（Execute）、存储结果（Store Result）。以下是每个阶段的具体过程。

1. 取指（Fetch）

目标：从内存中取出指令。

  • 步骤：

    1. 程序计数器（PC）的值通过总线送入 MAR（Memory Address Register）。
    2. 存储器将 PC 指向的内存单元中的数据送入 MDR（Memory Data Register）。
    3. MDR 的值通过总线写入 IR（Instruction Register）。
    4. PC 增加 1，PCMUX 选择 PC + 1，并通过控制信号 LD.PC 写回 PC。

  • 关键数据流：
    PC → MAR → Memory → MDR → IR

2. 解码（Decode）

目标：解析 IR 中的指令，生成控制信号。

  • 步骤：
  1. 从 IR 中提取：
    • 操作码（0001）表示这是 ADD 指令。
    • 源寄存器 SR1（R1）和 SR2（R3）。
    • 目标寄存器 DR（R2）。
  2. 控制单元生成以下控制信号：
    • 指定读取的寄存器（R1、R3）。
    • 设置 ALU 操作为加法（ALUK = ADD）。

3. 取操作数（Fetch Operands）

目标：从寄存器中获取操作数。

  • 步骤：
  1. SR1（R1） 的值通过 SR1.OUT 输出，并送入 ALU 的 A 输入端。
  2. SR2（R3） 的值通过 SR2MUX 直接送入 ALU 的 B 输入端。
  3. 控制信号 SR2MUX 指定从寄存器读取第二个操作数，而非立即数。
  • 关键数据流：
    R1 (SR1) → ALU.A R3 (SR2) → SR2MUX → ALU.B

4. 执行（Execute）

目标：在 ALU 中执行加法操作。

  • 步骤：
  1. ALU 接收 A 和 B 输入，并执行加法操作（R1 + R3）。
  2. 结果存储在 ALU 的输出寄存器中。
  • 关键数据流：
    ALU(A = R1, B = R3) → Result

5. 存储结果（Store Result）

目标：将 ALU 的运算结果存入目标寄存器 DR（R2）。

  • 步骤：
  1. ALU 的输出通过总线传递到寄存器组。
  2. 控制信号 LD.REG 置位，指示将数据写入 R2。
  • 关键数据流：
    ALU Result → Bus → R2

关键模块的使用

在这一过程中，以下模块发挥了重要作用：

  1. 寄存器组：读取源寄存器（R1、R3）的值，写回目标寄存器（R2）。
  2. ALU：执行加法运算。
  3. 控制单元：解析指令并生成对应的控制信号。
  4. 总线：在各模块之间传递数据。

立即数形式的 ADD 指令

若指令为 ADD R2, R1, #5（即 R2 = R1 + 5），数据通路会稍有不同：

  • SR2MUX 将从指令的立即数部分（#5）读取数据，而非寄存器 SR2。
  • SEXT 对立即数进行符号扩展后，送入 ALU 的 B 输入。

数据流变为：
R1 → ALU.A，#5 → SEXT → SR2MUX → ALU.B → ALU Result → Bus → R2
