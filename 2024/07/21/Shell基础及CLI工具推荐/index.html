Shell基础及CLI工具推荐

主要内容

  • 了解 Linux
  • 区分 Shell 与 Terminal
  • 掌握 Shell 基础命令，包括文件与目录操作
  • 了解 Shell 的进阶概念：重定向、管道等
  • 了解 vim 编辑器的基本用法，以及如何自行学习
  • 了解 GNU make 的基本用法，以及如何自行学习
  • 了解一些常用的命令行工具

主流计算机操作系统

  • Microsoft Windows (1985.11.20)
  • macOS (2001.3.24)
  • Linux (1991.9.17)

什么是Shell

是 Shell 吗？

  • 一个黑黑的窗口？
  • 一个输入奇怪命令的地方？
  • 一个看起来很高级的界面？

这些都不是Shell，这些所描述的窗口其实是Terminal，即终端。

什么是Terminal- 起源

  • 早期计算机没有桌面环境，唯一的交互方式是键盘
  • 用于输入数据，运行程序，并获得输出的“终端”

什么是Terminal- 现今

  • Terminal Emulator，模拟传统终端的行为
  • 一个应用程序，提供了一个窗口，和输入输出交互的功能
  • 内部运行的是Shell，Shell才是执行命令得到的东西

都有什么常见的Terminal呢？

  • Windows：Windows Terminal
  • Linux： Gnome Terminal、Konsole、LXTerminal等
  • macOS：Terminal.app、iTerm2等
  • 跨平台
    • Alacritty：基于Rust开发
    • Warp：基于Rust开发
    • Hyper：基于Electron开发
    • ……

So what is a Shell

  • “壳层”，也是一个程序，是用户与系统内核交互的界面
  • 负责接受并解析输入，交给更底层（操作系统）来执行，并返回输出

都有什么常见的Shell呢？

  • Windows下：cmd.exe、PowerShell 5
  • *nix下：
    • sh：Bourne Shell，最早、最经典的Shell
    • bash：Bourne Again Shell，最常用的shell
      • 大部分Linux发行版的默认shell
    • zsh：Z shell，功能强大、可高度自定义的shell
      • macOS Catalina起成为默认的shell
    • fish：
    • ……

Shell与Terminal

  • Terminal 从用户获取输入，然后传递给shell，等待shell处理完之后，将结果再传递回用户（显示在屏幕上）
  • shell从terminal拿到输入指令，解析后交给操作系统执行，然后将结果返回给terminal

一些例子：

  • nix系统下，用户默认的shell可以随意更换
  • Ctrl-c到底是复制还是中断程序？
    • 和shell无关，是terminal的行为
    • 比如macOS上iterm2就可以随意更改按键的这些行为
    • 如果定义了^c应该复制，那么terminal就会直接复制这些内容到剪贴板
    • 如果定义了^c应该中断程序，那么terminal就会告诉shell，shell再通过SIGHNT信号通知内核中断程序

为什么要使用命令行？它可以做什么？

  • 命令行是一种操作计算机的方式，理论上可以做任何事情
  • 可以让你手在键盘上就能控制电脑，而不需要鼠标点来点去
  • 在没有显示器的机器上(例如通过ssh连接到服务器)，只能通过命令行操作

甚至可以在命令行中查看网页（）

基础shell命令

Prompt能告诉我们什么

prompt即命令提示符，用来等待输入并给你提供一些信息，其中最重要的信息就是**工作路径(working directory)*，是当前shell所处的“位置”
一定要时时刻刻知道自己“在哪里”
举一个例子：命令 rm -rf ./ 它的作用是删除当前目录下的所有文件
通常还要有的信息是当前正在操作的用户，这个和权限有关，比如普通用户还是root等
向其中输入命令后回车，就可以执行命令

nix下的路径分隔符是/，Windows下是\
Windows下有多个“根目录”，即不同的“盘”，比如C盘，D盘，nix没有分盘概念，所有的文件都挂载在唯一的根目录/下。

路径相关命令

  • pwd：获取当前路径（print working directory）
  • cd -path：切换路径（change directory）
    path可以是“相对路径”或者“绝对路径”
    path中~代表home，.代表当前路径，..代表上一级路径

文件/目录操作命令

  • ls：列出当前路径下的文件和目录
    -a:列出所有文件和目录，包括隐藏文件
    -l：列出详细信息（文件权限，大小，修改时间等）
  • touch file：创建一个文件
  • mkdir dir：创建一个目录
  • cp src dst：复制文件或目录
    -r递归复制目录
  • mv src dst：移动文件或目录（重命名）
  • rm files…：删除文件
    -r：递归删除目录；-f：强制删除
  • find path -name pattern：在path下查找文件名匹配pattern的文件

文件内容查看命令

  • cat files…：输出与拼接文件
    -n：带行号输出
  • head file：输出file前10行
    -n lines：输出lines行
  • tail file：输出file后10行
    -n lines：输出lines行
  • more/less file：分页输出file的内容
    • 空格翻页，回车下一行，q退出
    • less的功能更多，比如查找，更好地翻页等，用法见 less –help
  • hexdump file：以十六进制输出文件内容
    • -C：并排输出十六进制与ASCII
    • -n N：输出前N个字节

其他命令

  • man：查看命令文档
  • echo:输出字符串（常配合重定向/管道使用）
  • whoami：获取当前用户
  • whereis/which/whence：查找命令所在位置
  • clear：清屏
  • chmod：更改文件权限
  • ps：显示进程信息
  • date：获取当前日期时间
  • kill：杀死进程
  • grep：搜索文件内容（常配合重定向/管道使用）
  • diff：比较文件/目录的内容
  • curl：发送HTTP请求；
  • wget：下载文件

重定向

重定向即文件流重定向。
有三种流：stdin标准输入流，stdout标准输出流，stderr标准错误流
通过重定向，可以更便捷地将输出存入文件或将文件内容作为程序输出。
通过> file将stdout重定向到文件，通过< ***file***将文件重定向到stdin
通过2 > file将stderr重定向到文件
两个右箭头>>表示追加模式，即不覆盖原文件，而是追加到文件末尾
通过& > file将stdout和stderr都重定向到文件
常见的用法有：

  • echo “hello world” > file:将字符串hello world写入file
  • cat file1 file2 > file3:将file1和file2拼接，输出到file3
  • diff file1 file2 > file3:将file1和file2的差异写入file3
  • ./a.out < file:将file作为a.out的输入

管道

通过管道可以将一个命令的输出作为另一个命令的输入，管道操作符|，将左侧命令的stdout连接到右侧命令的stdin。
通过管道可以将多个命令连接起来，形成一恶搞命令序列，可以通过一行命令来完成相对复杂的操作（e.g. SadServers Saskatoon）
例如：
ls | head -n 15 | tail -n 5
提取前十五个文件，再把这十五文件作为输入，提取后五个文件，那么效果上将就是list出十一个到十五个文件。
复杂命令看不懂，可以使用explainshell

环境变量

环境变量记录了系统信息的变量，可供程序读取
通过 echo $var来输出某一环境变量的值，env命令查看所有环境变量
绝大部分命令都是可执行文件，例如ls命令（/bin/ls）， 执行命令就是执行程序
shell可以只通过ls就找到/bin/ls这个可执行文件，原因在于它是在PATH环境变量中查找。
设置环境变量用export var=value，仅在当前执行的shell有效，想要持久化，需要写入配置文件中
删除环境变量用unset var。

vim基础用法

vi：由ex发展而来的老牌文本编辑器
vim是vi的升级版，功能更加强大，是一种基于命令行的全平台文本/代码编辑器。可以帮助我们脱离鼠标，只用键盘完成文本和代码编辑。
如果想要在windows上使用vim：
使用wsl，当然使用vscode的vim插件也是一个选择

也有自带教程，使用vimtutor命令。
