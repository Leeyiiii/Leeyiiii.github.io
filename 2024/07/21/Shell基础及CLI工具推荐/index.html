主流计算机操作系统

  • Microsoft Windows (1985.11.20)
  • macOS (2001.3.24)

什么是Shell

是 Shell 吗？

  • 一个黑黑的窗口？
  • 一个输入奇怪命令的地方？
  • 一个看起来很高级的界面？

这些都不是Shell，这些所描述的窗口其实是Terminal，即终端。

什么是Terminal- 起源

  • 早期计算机没有桌面环境，唯一的交互方式是键盘
  • 用于输入数据，运行程序，并获得输出的“终端”

什么是Terminal- 现今

  • Terminal Emulator，模拟传统终端的行为
  • 一个应用程序，提供了一个窗口，和输入输出交互的功能
  • 内部运行的是Shell，Shell才是执行命令得到的东西

都有什么常见的Terminal呢？

  • Windows：Windows Terminal
  • Linux： Gnome Terminal、Konsole、LXTerminal等
  • macOS：Terminal.app、iTerm2等
  • 跨平台
    • Alacritty：基于Rust开发
    • Warp：基于Rust开发
    • Hyper：基于Electron开发
    • ……

So what is a Shell

  • “壳层”，也是一个程序，是用户与系统内核交互的界面
  • 负责接受并解析输入，交给更底层（操作系统）来执行，并返回输出

都有什么常见的Shell呢？

  • Windows下：cmd.exe、PowerShell 5
  • *nix下：
    • sh：Bourne Shell，最早、最经典的Shell
    • bash：Bourne Again Shell，最常用的shell
      • 大部分Linux发行版的默认shell
    • zsh：Z shell，功能强大、可高度自定义的shell
      • macOS Catalina起成为默认的shell
    • fish：
    • ……

Shell与Terminal

  • Terminal 从用户获取输入，然后传递给shell，等待shell处理完之后，将结果再传递回用户（显示在屏幕上）
  • shell从terminal拿到输入指令，解析后交给操作系统执行，然后将结果返回给terminal

一些例子：

  • nix系统下，用户默认的shell可以随意更换
  • Ctrl-c到底是复制还是中断程序？
    • 和shell无关，是terminal的行为
    • 比如macOS上iterm2就可以随意更改按键的这些行为
    • 如果定义了^c应该复制，那么terminal就会直接复制这些内容到剪贴板
    • 如果定义了^c应该中断程序，那么terminal就会告诉shell，shell再通过SIGHNT信号通知内核中断程序

为什么要使用命令行？它可以做什么？

  • 命令行是一种操作计算机的方式，理论上可以做任何事情
  • 可以让你手在键盘上就能控制电脑，而不需要鼠标点来点去
  • 在没有显示器的机器上(例如通过ssh连接到服务器)，只能通过命令行操作
